### Best Practices

1. **Keep sensitive data out of Git**: 
   - Never commit connection strings with passwords
   - Use User Secrets for development: `dotnet user-secrets set "ConnectionStrings:DefaultConnection" "your-connection-string"`

2. **Branch Strategy**:
   - `main` - Production-ready code
   - `develop` - Development branch
   - `feature/*` - Feature branches

3. **Commit Messages**:
   - Use clear, descriptive commit messages
   - Follow conventional commits: `feat:`, `fix:`, `docs:`, `test:`, `refactor:`

4. **Documentation**:
   - Keep README.md updated
   - Document API changes
   - Maintain COVERAGE.md

5. **Testing**:
   - Write tests for new features
   - Maintain 80%+ coverage
   - Run tests before committing
